# TransPlayer 产品需求文档（PRD）

**版本**: v1.0  
**创建日期**: 2025  
**最后更新**: 2025

---

## 1. 项目概述

### 1.1 项目信息
- **项目名称**: TransPlayer（即时字幕播放器）
- **项目定位**: 支持实时字幕生成和翻译的Android视频播放器
- **核心价值**: 提供完全离线的视频播放、字幕生成和翻译功能，保护用户隐私

### 1.2 核心特性
- ✅ 本地视频文件播放
- ✅ 网络流媒体播放
- ✅ 实时本地语音识别生成字幕
- ✅ 实时本地翻译字幕
- ✅ 完全离线运行
- ✅ 字幕管理和导出

### 1.3 目标用户
- 需要观看外语视频并理解内容的用户
- 听力障碍用户
- 学习外语的用户
- 注重隐私、不希望数据上传的用户

## 2. 功能需求详细规格

### 2.1 视频播放功能

#### 2.1.1 视频源支持
**需求描述**: 应用应支持多种视频来源

**功能点**:
- **本地视频文件**
  - 支持格式: MP4, MKV, AVI, MOV, FLV, 3GP, WEBM
  - 支持从系统文件管理器选择
  - 支持从最近文件列表快速访问
  - 文件大小限制: 无硬性限制，但需优化大文件处理
  
- **网络流媒体**
  - 支持HTTP/HTTPS视频URL
  - 支持HLS (m3u8)流媒体
  - 支持DASH流媒体
  - 支持视频URL输入界面
  - 网络错误处理和重试机制

#### 2.1.2 播放控制功能
**需求描述**: 提供完整的视频播放控制能力

**功能点**:
- **基础控制**
  - 播放/暂停按钮
  - 进度条拖拽（支持精确跳转）
  - 快进/快退（10秒、30秒、1分钟）
  - 音量控制（系统音量 + 应用内音量）
  - 播放速度调节（0.5x, 0.75x, 1.0x, 1.25x, 1.5x, 2.0x）
  
- **显示控制**
  - 全屏/退出全屏
  - 屏幕方向锁定
  - 亮度调节（全屏模式下）
  - 画面比例调整（16:9, 4:3, 原始比例, 填充屏幕）

#### 2.1.3 播放历史
**需求描述**: 记录用户观看历史，支持断点续播

**功能点**:
- 自动记录播放进度
- 播放历史列表（最近10个视频）
- 断点续播功能
- 清除历史记录功能

#### 2.1.4 性能要求
- 视频加载时间 < 3秒（本地文件）
- 网络视频缓冲时间 < 5秒（良好网络条件下）
- 播放流畅度: 60fps（支持帧率）
- 内存占用: < 200MB（不含模型文件）

---

### 2.2 实时字幕生成功能

#### 2.2.1 语音识别引擎
**需求描述**: 使用本地语音识别引擎实时生成字幕

**技术方案**:
- **主要方案**: ML Kit Speech Recognition（离线模式）
  - 支持语言: 中文、英文、日文、韩文等（根据ML Kit支持）
  - 离线模型大小: 约50-100MB
  - 识别延迟: < 2秒
  
- **备选方案**: TensorFlow Lite自定义模型
  - 如果ML Kit不支持目标语言或性能不满足需求
  - 需要自行训练或获取模型

#### 2.2.2 字幕生成流程
**需求描述**: 从视频音频中提取并识别语音，生成时间轴同步的字幕

**功能点**:
- **音频提取**
  - 从视频中实时提取音频流
  - 音频采样率: 16kHz（ML Kit推荐）
  - 音频格式: PCM 16-bit
  
- **实时识别**
  - 连续语音识别（非单次识别）
  - 自动分段（基于静音检测）
  - 识别结果实时显示
  
- **时间轴同步**
  - 字幕与视频时间轴精确对齐
  - 字幕显示时长: 根据语音长度自动调整（最小2秒，最大10秒）
  - 字幕重叠处理（新字幕出现时旧字幕淡出）

#### 2.2.3 字幕显示
**需求描述**: 在视频画面上显示生成的字幕

**功能点**:
- **显示位置**
  - 默认位置: 视频底部（距离底部10%）
  - 可调整位置: 顶部、中部、底部
  - 字幕区域宽度: 视频宽度的80%（居中）
  
- **字幕样式**
  - 字体大小: 可调节（12sp - 24sp）
  - 字体颜色: 可自定义（默认白色）
  - 背景: 半透明黑色背景（可选）
  - 描边: 文字描边（可选，提高可读性）
  - 字体: 系统默认字体或自定义字体
  
- **显示控制**
  - 字幕开关（显示/隐藏）
  - 字幕透明度调节
  - 字幕延迟显示（可选，用于同步调整）

#### 2.2.4 字幕数据管理
**需求描述**: 管理生成的字幕数据

**功能点**:
- 字幕数据存储（时间轴 + 文本）
- 字幕自动保存（与视频文件关联）
- 字幕加载（播放相同视频时自动加载）
- 字幕清除功能

#### 2.2.5 性能要求
- 识别延迟: < 2秒（从语音到字幕显示）
- CPU占用: < 30%（在中等性能设备上）
- 内存占用: < 100MB（识别引擎）
- 准确率: > 85%（清晰语音条件下）

---

### 2.3 字幕翻译功能

#### 2.3.1 翻译引擎
**需求描述**: 使用本地翻译模型实时翻译字幕

**技术方案**:
- **主要方案**: TensorFlow Lite翻译模型
  - 模型类型: 神经机器翻译（NMT）
  - 支持语言对: 
    - 中英文互译（必需）
    - 日文、韩文、法文、德文等（可选）
  - 模型大小: 约50-200MB/语言对
  - 翻译延迟: < 1秒
  
- **模型来源**:
  - Google Translate TFLite模型
  - 或开源翻译模型（如OpenNMT）
  - 或自行训练的模型

#### 2.3.2 翻译功能
**需求描述**: 将识别出的字幕实时翻译为目标语言

**功能点**:
- **实时翻译**
  - 字幕生成后自动触发翻译
  - 翻译结果实时显示
  - 支持显示原文和译文（双语模式）
  
- **语言选择**
  - 源语言: 自动检测或手动选择
  - 目标语言: 用户选择
  - 语言切换: 播放过程中可切换
  - 语言选择界面: 清晰的列表选择
  
- **翻译显示模式**
  - 仅原文
  - 仅译文
  - 双语显示（原文在上，译文在下）
  - 双语显示（原文和译文并排）

#### 2.3.3 翻译质量优化
**需求描述**: 提升翻译准确性和流畅度

**功能点**:
- 上下文感知翻译（考虑前后字幕）
- 专业术语识别（可选词典）
- 翻译结果缓存（相同文本不重复翻译）
- 用户反馈机制（标记翻译错误，用于改进）

#### 2.3.4 性能要求
- 翻译延迟: < 1秒（单句字幕）
- CPU占用: < 20%（翻译模型推理）
- 内存占用: < 150MB（翻译模型）
- 翻译准确率: > 75%（日常对话场景）

### 2.4 离线功能

#### 2.4.1 离线运行要求
**需求描述**: 所有核心功能必须在无网络环境下正常工作

**功能点**:
- 视频播放: 完全离线（本地文件）
- 字幕生成: 完全离线（本地模型）
- 字幕翻译: 完全离线（本地模型）
- 离线模式检测和提示

#### 2.4.2 本地模型管理
**需求描述**: 管理语音识别和翻译所需的本地模型文件

**功能点**:
- **模型下载**
  - 首次启动时提示下载模型
  - 模型下载进度显示
  - 支持暂停/恢复下载
  - 支持后台下载
  
- **模型存储**
  - 存储位置: 应用私有目录
  - 模型文件加密（可选）
  - 模型版本管理
  
- **模型更新**
  - 检查模型更新（可选，需要网络）
  - 增量更新支持
  - 更新失败回退机制
  
- **模型管理界面**
  - 已安装模型列表
  - 模型大小显示
  - 模型删除功能
  - 模型下载状态

#### 2.4.3 存储空间管理
**需求描述**: 优化模型文件占用空间

**功能点**:
- 模型文件总大小: < 500MB（所有语言）
- 按需下载（用户选择语言后再下载）
- 未使用模型清理建议
- 存储空间不足提示

---

### 2.5 字幕管理功能

#### 2.5.1 字幕导出
**需求描述**: 将生成的字幕导出为标准格式文件

**功能点**:
- **支持格式**
  - SRT (SubRip)
  - VTT (WebVTT)
  - ASS (Advanced SubStation Alpha) - 可选
  
- **导出内容**
  - 时间轴信息
  - 字幕文本（原文）
  - 字幕文本（译文，如果启用翻译）
  - 字幕样式信息（如果格式支持）
  
- **导出方式**
  - 保存到本地文件
  - 分享到其他应用
  - 导出位置选择

#### 2.5.2 字幕编辑
**需求描述**: 允许用户手动编辑生成的字幕

**功能点**:
- **编辑功能**
  - 修改字幕文本
  - 调整字幕时间轴
  - 添加/删除字幕条目
  - 合并/拆分字幕
  
- **编辑界面**
  - 字幕列表视图
  - 时间轴编辑器
  - 文本编辑器
  - 预览功能

#### 2.5.3 字幕保存和加载
**需求描述**: 保存编辑后的字幕，并在播放时加载

**功能点**:
- 自动保存（编辑后）
- 手动保存
- 字幕文件关联（与视频文件）
- 播放时自动加载关联字幕
- 字幕文件管理（查看、删除）

## 3. 技术架构详细设计

### 3.1 开发环境

#### 3.1.1 平台要求
- **目标平台**: Android
- **最低SDK版本**: API 26 (Android 8.0 Oreo)
- **目标SDK版本**: API 34 (Android 14)
- **编译SDK版本**: API 34
- **Gradle版本**: 8.0+

#### 3.1.2 开发工具
- **开发语言**: Kotlin 100%
- **IDE**: Android Studio Hedgehog+
- **构建工具**: Gradle
- **版本控制**: Git

#### 3.1.3 架构模式
- **架构**: MVVM (Model-View-ViewModel) + Clean Architecture
- **分层结构**:
  ```
  Presentation Layer (UI)
    ├── Compose UI
    └── ViewModels
  Domain Layer (Business Logic)
    ├── Use Cases
    └── Domain Models
  Data Layer
    ├── Repositories
    ├── Data Sources (Local/Remote)
    └── Models
  ```

### 3.2 核心技术栈

#### 3.2.1 视频播放
- **ExoPlayer 2.19+**
  - 支持多种视频格式
  - 支持网络流媒体
  - 可扩展的架构
  - 自定义渲染器支持
  
- **功能模块**:
  - `PlayerModule`: ExoPlayer实例管理
  - `VideoPlayerView`: 自定义播放器视图
  - `PlaybackController`: 播放控制逻辑

#### 3.2.2 语音识别
- **ML Kit Speech Recognition (离线模式)**
  - 依赖: `com.google.mlkit:speech-recognition`
  - 离线模型自动下载
  - 支持连续识别
  
- **备选方案**: TensorFlow Lite
  - 依赖: `org.tensorflow:tensorflow-lite`
  - 自定义模型加载
  - 需要自行实现音频预处理

- **功能模块**:
  - `SpeechRecognitionModule`: 识别引擎封装
  - `AudioExtractor`: 音频提取
  - `SubtitleGenerator`: 字幕生成逻辑

#### 3.2.3 翻译引擎
- **TensorFlow Lite**
  - 依赖: `org.tensorflow:tensorflow-lite`
  - 模型文件: `.tflite`格式
  - GPU加速支持（可选）
  
- **功能模块**:
  - `TranslationModule`: 翻译引擎封装
  - `TranslationModelManager`: 模型管理
  - `TranslationCache`: 翻译结果缓存

#### 3.2.4 UI框架
- **Jetpack Compose**
  - 版本: Compose BOM 2024.01.00+
  - Material Design 3
  - 动画和过渡效果
  
- **关键组件**:
  - `VideoPlayerScreen`: 播放器主界面
  - `SubtitleOverlay`: 字幕叠加层
  - `SettingsScreen`: 设置界面
  - `ModelManagementScreen`: 模型管理界面

#### 3.2.5 依赖注入
- **Hilt**
  - 版本: 2.48+
  - 模块化依赖注入
  - ViewModel注入支持

#### 3.2.6 异步处理
- **Kotlin Coroutines**
  - 版本: 1.7.3+
  - 协程作用域管理
  - Flow用于数据流
  
- **关键使用场景**:
  - 视频加载
  - 字幕生成（后台处理）
  - 翻译处理（后台处理）
  - 模型下载

#### 3.2.7 数据存储
- **Room Database**
  - 版本: 2.6.1+
  - 播放历史存储
  - 字幕数据存储
  - 用户设置存储
  
- **DataStore**
  - 版本: 1.0.0+
  - 用户偏好设置
  - 应用配置

#### 3.2.8 网络请求
- **Retrofit + OkHttp**
  - 版本: Retrofit 2.9+, OkHttp 4.12+
  - 仅用于模型下载（可选）
  - 网络视频URL验证

### 3.3 本地模型管理

#### 3.3.1 模型文件存储策略
- **存储位置**: `/data/data/com.transplayer.app/files/models/`
- **目录结构**:

#### 3.3.2 模型版本管理
- **版本信息存储**: JSON格式元数据文件
- **版本检查**: 启动时检查模型完整性
- **版本更新**: 增量更新支持

#### 3.3.3 模型下载机制
- **下载方式**: 
  - 应用内下载（如果提供模型服务器）
  - 或首次安装时打包基础模型
- **下载进度**: 实时显示下载百分比
- **断点续传**: 支持下载中断后继续
- **校验机制**: MD5/SHA256校验确保文件完整性

## 4. 用户体验设计

### 4.1 界面设计规范

#### 4.1.1 设计系统
- **Material Design 3**
  - 动态颜色支持
  - 圆角、阴影、 elevation
  - 响应式布局
  
- **主题支持**
  - 浅色主题（Light Theme）
  - 深色主题（Dark Theme）
  - 跟随系统设置

#### 4.1.2 播放器界面布局

**主播放器界面**:
```
┌─────────────────────────────┐
│  [返回]        [设置] [更多] │
│                             │
│                             │
│        视频播放区域          │
│      (ExoPlayer Surface)     │
│                             │
│                             │
│  ┌───────────────────────┐ │
```

**全屏模式**:
- 隐藏系统UI

- **位置**: 视频底部，距离底部10%

  - 白色文字，黑色半透明背景
  - 圆角背景

- **播放设置**: 播放速度、自动播放下一集
- **字幕设置**: 字体大小、颜色、位置、样式
- **翻译设置**: 目标语言、显示模式
- **模型管理**: 已安装模型、下载新模型
- **关于**: 版本信息、帮助、反馈

### 4.2 交互流程设计

#### 4.2.1 视频选择流程
```
启动应用
  ↓
主界面（视频列表/文件选择）
  ↓
选择视频（本地文件/输入URL）
  ↓
检查模型（语音识别模型）
  ↓
[模型未安装] → 提示下载 → 下载完成
  ↓
开始播放
```

#### 4.2.2 字幕生成启动流程
```
播放视频
  ↓
用户点击"字幕"按钮
  ↓
检查语音识别模型
  ↓
[模型未安装] → 下载模型
  ↓
选择识别语言
  ↓
开始音频提取和识别
  ↓
实时显示字幕
```

#### 4.2.3 翻译功能启动流程
```
字幕已生成/正在生成
  ↓
用户点击"翻译"按钮
  ↓
选择目标语言
  ↓
检查翻译模型
  ↓
[模型未安装] → 下载模型
  ↓
开始实时翻译
  ↓
显示翻译结果
```

#### 4.2.4 字幕编辑流程
```
播放界面
  ↓
点击"编辑字幕"
  ↓
进入字幕编辑界面
  ↓
显示字幕列表（时间轴 + 文本）
  ↓
选择字幕条目
  ↓
编辑文本/时间
  ↓
保存更改
  ↓
返回播放界面
```

### 4.3 用户反馈和提示

#### 4.3.1 加载状态
- 视频加载: 显示加载进度条
- 模型下载: 显示下载进度对话框
- 字幕生成: 显示"正在生成字幕..."提示

#### 4.3.2 错误提示
- 网络错误: "网络连接失败，请检查网络设置"
- 视频加载失败: "无法加载视频，请检查文件或URL"
- 识别失败: "语音识别失败，请重试"
- 翻译失败: "翻译失败，请检查模型是否已安装"

#### 4.3.3 成功提示
- 字幕生成完成: Toast提示"字幕生成完成"
- 翻译完成: 自动显示，无需额外提示
- 字幕保存成功: Toast提示"字幕已保存"

## 5. 性能要求详细指标

### 5.1 响应时间
- **应用启动时间**: < 2秒（冷启动）
- **视频加载时间**: 
  - 本地文件: < 3秒
  - 网络视频: < 5秒（良好网络）
- **字幕生成延迟**: < 2秒（从语音到显示）
- **翻译响应时间**: < 1秒（单句字幕）

### 5.2 流畅度
- **视频播放帧率**: 60fps（支持帧率）
- **UI渲染帧率**: 60fps
- **字幕更新频率**: 实时（无卡顿）

### 5.3 资源占用
- **内存占用**:
  - 基础应用: < 100MB
  - 播放视频: +50MB
  - 语音识别: +100MB
  - 翻译模型: +150MB
  - 总计峰值: < 400MB
  
- **CPU占用**:
  - 空闲状态: < 5%
  - 播放视频: 10-20%
  - 生成字幕: +20-30%
  - 翻译字幕: +15-20%
  - 总计峰值: < 70%（中等性能设备）
  
- **存储空间**:
  - 应用安装包: < 50MB
  - 模型文件: < 500MB（所有语言）
  - 用户数据: 动态增长

### 5.4 电池续航
- **播放视频**: 与标准播放器相当
- **生成字幕**: 额外耗电 < 20%
- **翻译字幕**: 额外耗电 < 15%
- **优化策略**: 
  - 后台处理使用WorkManager
  - 模型推理使用GPU加速（如果可用）
  - 智能降频（低电量模式）

## 6. 非功能需求

### 6.1 隐私保护
- **数据本地处理**: 所有语音识别和翻译在设备本地完成
- **无数据上传**: 不向服务器发送任何用户数据
- **权限最小化**: 仅请求必要的系统权限
  - 存储权限（读取视频文件）
  - 网络权限（播放网络视频，下载模型）
  - 麦克风权限（如果需要实时录音，否则不需要）

### 6.2 安全性
- **模型文件校验**: 下载后验证文件完整性
- **代码混淆**: 使用ProGuard/R8混淆
- **安全存储**: 敏感数据加密存储

### 6.3 兼容性
- **Android版本**: 支持Android 8.0+
- **设备性能**: 
  - 最低要求: 3GB RAM, 四核CPU（Android 8.0+设备通常满足）
  - 推荐配置: 4GB+ RAM, 八核CPU
- **屏幕尺寸**: 支持手机和平板
- **屏幕方向**: 支持横屏和竖屏

### 6.4 可维护性
- **代码规范**: 遵循Kotlin编码规范
- **文档完善**: 关键模块添加注释
- **单元测试**: 核心业务逻辑单元测试覆盖率 > 60%
- **UI测试**: 关键用户流程UI测试

### 6.5 可扩展性
- **模块化设计**: 便于添加新功能
- **插件化架构**: 支持第三方翻译模型
- **多语言支持**: UI支持多语言（中英文）

### 6.6 错误处理
- **异常捕获**: 全局异常处理机制
- **错误日志**: 记录错误信息（本地，不上传）
- **用户友好提示**: 错误信息清晰易懂
- **恢复机制**: 关键操作失败后提供重试
